
开发一个极致轻量的跨平台校园网守护程序

**【项目定位】**
我需要你用 **Rust** 开发一个名为 `ePortal Guard` 的程序。它的目标是：全自动后台巡检校园网状态，发现掉线后自动执行用户提供的 cURL 命令。该程序必须追求**极致的轻量化**（目标运行内存 < 5MB，二进制文件 < 1MB），并且**严禁使用**任何重量级库。

**【核心逻辑与跨平台交互】**

1. **配置文件系统**：
* 自动识别 OS 路径：Windows (`AppData/Roaming`), macOS (`Library/Application Support`), Linux (`~/.config`)。
* 启动时若无配置，自动生成 `config.toml`（配置 ping 间隔、服务器、Web 端口）和 `curl.txt`（存放 cURL 命令）。


2. **无界面交互方案**：
* **托盘图标**：使用 `tray-icon` 实现跨平台托盘。菜单需包含：显示状态、手动登录、打开配置文件（打开config）、获取cURL教程（打开一个github网页链接）、粘贴cURL（打开curl.txt）、切换开机自启、退出程序。
* **Web 后门**：使用 `tiny_http` 开启一个极简本地服务器。当没有托盘的 GNOME 用户找不到托盘时，通过浏览器访问 `localhost:端口` 来实现托盘的所有功能。
* **文件联动**：当用户在 UI 或 Web 点击“编辑配置”时，调用系统默认编辑器（如记事本、TextEdit）直接打开 `curl.txt`。
* **系统通知**：调用各个系统以及桌面环境自带的系统通知来显示软件通知



**【技术细节要求】**

1. **后台监控**：开启一个独立线程，根据 `config.toml` 设定的频率（默认 5s）执行 `ping`。仅在 `ping` 失败且检测到本地拥有内网 IP 时，才调用系统 Shell 执行 `curl.txt` 里的命令，默认ping阿里云服务器。
2. **轻量化网络执行**：不要集成巨大的 `reqwest` 库。请直接使用 `std::process::Command` 调用系统自带的 `curl` 二进制文件，这样能最精准地执行用户粘贴的原始 cURL 字符串，对于没有curl命令的系统，用系统通知返回一个报错。
3. **自启实现**：
* Windows：通过修改注册表 `Run` 键值。
* macOS：生成并管理 `LaunchAgents` 的 `.plist` 文件。
* Linux：提供一个简单的 `.desktop` 文件生成逻辑。
4. **系统通知**：使用 `notify-rust`。在“成功登录”、“程序退出”以及“配置更新”或者软件有什么报错的时候发送通知

**【编译优化指令】**
请在 `Cargo.toml` 中配置 `panic = "abort"`, `lto = true`, `opt-level = "z"`, `codegen-units = 1`，并确保 Windows 编译时使用 `#![windows_subsystem = "windows"]` 隐藏命令行黑窗口。

**【额外要求】**
请务必使用 Rust 的 cfg 属性宏 和 模块化设计 来处理跨平台差异（如路径获取、自启动实现和 Shell 调用），确保同一套源代码可以通过不同的 target 参数编译出适配各个系统的独立程序，目的仍然是保证二进制文件体积小

**【开发步骤】**

0. 开始前，配置开发环境和git仓库
1. 首先，搭建跨平台文件路径映射和单实例文件锁逻辑。
2. 第二步，实现基于 `tiny_http` 的极简 Web 路由（处理基本的 GET 查看和 POST 存值）。
3. 第三步，实现核心的后台 `ping` 巡检线程与 `curl` 执行器。
4. 第四步，集成托盘图标和系统通知，将所有模块串联。

请按照上述要求和步骤开发这个软件
